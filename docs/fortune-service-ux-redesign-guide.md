# Fortune Service UX 재설계 가이드 (실서비스 기준)

## 1) 문서 목적
현재 개발 UI(임시 테스트용)를 실서비스 수준으로 재설계하기 위한 기준 문서입니다.  
화면별 목적, 필수 요소, 상태, 오류/예외, 핵심 액션을 일관된 UX 규칙으로 정리합니다.

## 2) 디자인 방향
- 키워드: 세련됨, 깔끔함, 높은 가독성, 명확한 상태 전달
- 원칙:
  - 정보 위계 우선: 한 화면에서 사용자가 “지금 무엇을 해야 하는지” 3초 내 이해
  - 상태 가시성: 로딩/성공/빈화면/에러를 동일 패턴으로 제공
  - 입력 신뢰성: 출생정보처럼 민감하고 복잡한 입력은 실시간 검증 + 명확한 안내
  - 예외 명확성: 결제/구독 상태(실패/만료/유예)는 배지와 메시지로 즉시 구분

## 3) 화면별 UX 요구사항

| 화면 | 목적 | 필수 요소 | 주요 상태 | 오류/예외 | 액션 |
|---|---|---|---|---|---|
| 온보딩 | 서비스 가치/주의사항 전달 | 핵심 가치 3~4포인트, 참고용 고지, 시작 버튼 | 최초 진입, 재진입 스킵 | 네트워크 없이도 표시 가능 | 시작하기, 로그인으로 이동 |
| 로그인 | 사용자 인증 | 이메일, 비밀번호, 로그인 버튼, 회원가입/비번찾기 링크 | 입력 전/입력 완료/로딩 | 로그인 실패, 계정 없음, 비밀번호 오류 | 로그인, 회원가입 이동 |
| 출생정보 입력 | 사주 계산 입력 수집 | 생년월일, 출생시간(미상 토글), 출생지, 성별, 양력/음력, 윤달 | 신규 작성, 기존 수정, 저장 완료 | 필수값 누락, 음력+윤달 불일치, 날짜 범위 오류 | 저장, 다음(결과로) |
| 출생프로필 선택 | 기존 프로필 재사용 | 프로필 리스트(날짜/지역/달력종류), 선택 상태, 새로 만들기 버튼 | 목록 있음/없음 | 목록 조회 실패, 권한 문제 | 프로필 선택, 생성 이동 |
| 결과 대시보드 | 계산 결과 요약 노출 | 사주 4주(연월일시), 오행 분포 시각화, 핵심 요약 카드 | 로딩/성공 | 계산 실패, 데이터 없음 | 상세 리포트 이동, 오늘 운세 이동 |
| 상세 리포트 | 주제별 해석 제공 | 탭(성향/연애/직업), 요약/강점/주의/행동가이드 | 탭 전환, 로딩 | 리포트 생성 실패 | 리포트 생성/재시도 |
| 오늘 운세 | 일일 액션 제공 | 점수, 카테고리(연애/일/재물/건강), 오늘 액션 3개 | 오늘 데이터 있음/없음 | 생성 실패, 날짜 오류 | 오늘 운세 생성/새로고침 |
| 리포트 목록 | 저장 결과 확인 | 리포트 리스트(유형/생성시간/표시여부), 필터 | 데이터 있음/없음 | 조회 실패 | 항목 상세 이동 |
| 결제/주문 | 결제 상태 확인 | 주문 리스트(상태/생성일/상품), 상태 배지 | 결제 전/완료/실패 | 조회 실패 | 주문 상세 이동 |
| 구독 관리 | 구독 상태 확인 | 플랜명, 상태(active/grace/expired/canceled), 시작/만료일 | 구독 있음/없음 | 조회 실패 | 구독 관리/해지 이동 |
| 마이페이지 | 계정/설정 허브 | 계정 정보, 출생정보 관리, 주문/구독 진입, 약관/개인정보 | 일반 | 로그아웃 실패(희박) | 로그아웃, 각 메뉴 이동 |
| 공통 로딩/에러 | 일관된 피드백 | 스켈레톤/로딩 인디케이터, 토스트/에러박스, 재시도 버튼 | 로딩/빈화면/에러 | requestId 노출(문의용) | 재시도, 문의하기 |

## 4) 핵심 4개 화면 정보 위계 (최우선)

### A. 출생정보 입력
- 1순위: 계산 필수값(생년월일, 달력종류, 성별)
- 2순위: 정확도 향상값(출생시간, 출생지, 윤달)
- 3순위: 안내/검증 메시지(오류, 보정 힌트)
- UI 권장:
  - 단계형 섹션 구분: 기본정보 → 시간/장소 → 캘린더 옵션
  - 실시간 유효성 검사(필드 하단), 제출 시 상단 요약 에러 박스 동시 제공
  - “출생시간 미상” 토글 시 관련 입력 비활성 + 설명 문구 노출

### B. 결과 대시보드
- 1순위: 사주 핵심 결과(4주: 연/월/일/시)
- 2순위: 오행 분포(시각화 + 한줄 해석)
- 3순위: 핵심 요약 카드(성향/주의점/추천 행동)
- UI 권장:
  - 첫 화면 상단에 “요약 결론”을 카드 1~2개로 압축
  - 차트는 시각요소 + 텍스트 수치 병기(해석 가능성 강화)
  - 하단 고정 CTA: `상세 리포트 보기`, `오늘 운세 보기`

### C. 상세 리포트
- 1순위: 탭 구조(성향/연애/직업) + 현재 탭 명확 표시
- 2순위: 탭별 요약(3~5줄)
- 3순위: 강점/주의/행동가이드(실행 가능한 문장)
- UI 권장:
  - 탭 전환 시 스켈레톤 유지, 레이아웃 점프 최소화
  - 긴 텍스트는 “요약 → 펼치기” 구조로 피로도 감소
  - 행동가이드는 체크리스트형 문장으로 구체화

### D. 오늘 운세
- 1순위: 오늘 점수(총점/한줄 총평)
- 2순위: 카테고리 점수(연애/일/재물/건강)
- 3순위: 오늘 액션 3개(즉시 실행 가능)
- UI 권장:
  - “오늘” 기준 날짜를 명시하고 시간대 혼란 방지
  - 데이터 없을 때는 생성 유도형 빈화면 제공
  - 새로고침 시 마지막 갱신 시각 표시

## 5) 상태/예외 표현 규칙

### 로딩
- 첫 진입: 스켈레톤 중심
- 버튼 액션: 버튼 내부 스피너 + 중복 클릭 방지

### 빈화면
- 원인 설명 1문장 + 다음 행동 1개(주요 CTA)

### 에러
- 사용자 문구 + `requestId` + 재시도 버튼
- 치명/비치명 분리:
  - 치명: 전체 에러 박스
  - 비치명: 인라인 경고 + 부분 재시도

## 6) 결제/구독 상태 배지 기준
- 결제:
  - `결제 완료`(Success)
  - `결제 대기`(Pending)
  - `결제 실패`(Failed)
- 구독:
  - `active`(이용중)
  - `grace`(유예기간)
  - `expired`(만료)
  - `canceled`(해지)
- 규칙:
  - 텍스트만으로도 의미 전달되게 라벨 명확화
  - 실패/만료/유예는 동일 강도의 강조(배지 + 보조 설명)
  - 상세 화면에서 원인/해결 액션(재결제, 결제수단 변경, 문의) 제공

## 7) 디자이너 전달 문구 (원문 반영)
- “현재 개발 UI는 임시 테스트용이며, 위 표의 화면/상태 기준으로 실서비스 UX를 재설계 부탁드립니다.”
- “특히 출생정보 입력, 결과 대시보드, 상세 리포트, 오늘 운세 4개 화면의 정보 위계와 가독성을 최우선으로 잡아주세요.”
- “결제/구독 화면은 상태 배지와 예외 상태(실패/만료/유예)를 명확히 구분해 주세요.”

## 8) 산출물 요청 포맷 (디자인팀 공유용)
- 화면별 와이어프레임(기본/로딩/빈화면/에러)
- 컴포넌트 스펙(입력폼, 배지, 카드, 탭, 에러박스)
- 상태 전이 플로우(로그인, 출생정보 저장, 리포트 생성, 운세 생성)
- QA 체크리스트(오류 문구, 예외 케이스, 접근성, 가독성)
